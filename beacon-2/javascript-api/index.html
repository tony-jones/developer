Beacon JS API

The Beacon JS API exposes the following methods to the global object Beacon. You can use them to control your Beacon programmatically.

Beacon(‘init’, beaconId)

Call this method to render the Beacon. You can call it with a string or an object with initial configuration:

Beacon('init', '0c09af96-546a-4e40-a797-e747d74d5437')

Beacon(‘destroy’)

Unmounts the Beacon from the DOM. You can call ‘init’ again to re-render it.

Beacon(‘search’, query)

Makes a search operation and navigates to the search results screen:

Beacon('search', 'hipaa')

Beacon(‘navigate’, route)

Navigates to a specific screen. Available routes are:

Beacon('navigate', '/') // welcome screen
Beacon('navigate', '/ask/message/') // message screen
Beacon('navigate', '/docs/search?query=help') // === Beacon('search')

Use these methods to manually open or close the Beacon

Beacon('open')
Beacon('close')
Beacon('toggle')

Beacon(‘identify’, userObject)

The identify method serves three purposes:

  1. It tells the Beacon that we know who the visitor is (at least his email address), and we use their name and email address to prepopulate and hide these fields in the contact form, or, in the future, skip these questions during the chat flow.
  2. It can be used to track custom data attributes that are then synced with the customer’s profile in Help Scout and displayed on the Beacon Sidebar app.
  3. It can be used to authenticate a user in “secure mode” and retrieve their previous messages if a signature attribute is provided.

Example usage:

Identify a user in normal mode:

Beacon('identify', {
  name: 'Steve Aoki',
  email: 'steve@aoki.com'
})

Identify a user in secude mode:

Beacon('identify', {
  name: 'Steve Aoki',
  email: 'steve@aoki.com',
  signature: '2fe4d4a5963f28b77737c091c436096beee0b74fabb9fcdcd2a4d8859d2099a3'
})

Identify a user in secure mode and log custom attributes to the Beacon Sidebar app:

Beacon('identify', {
  name: 'Steve Aoki',
  email: 'steve@aoki.com',
  signature: '2fe4d4a5963f28b77737c091c436096beee0b74fabb9fcdcd2a4d8859d2099a3',
  'Subscription': 'Free Plan',
  'Favorite Movie': 'Zoolander'
})

Beacon(‘prefill’, formObject)

This method can be used to pre-populate the message contact form with data, you can fill any of the fields: name, email, subject, body of the message (text), and custom fields. Unlike the identify method, the name and email fields will remain visible and editable when you use the prefill method.

Beacon('prefill', {
  name: 'Steve Aoki',
  email: 'steve@aoki.com',
  subject: 'Need help with invoice',
  text: 'Hello, I need some help with my invoice. See attached PDF...',
  fields: [{
    id: 5678,
    value: 'Question'
  }]
})

Beacon(‘reset’)

Resets the contact form by clearing all of its fields, custom fields and attachments.

These methods allow you to hook up to specific events in the lifecycle of the embed to perform custom actions. Right now the only two supported events are ‘open’ and ‘close’, and they’ll be executed when the Beacon is opened or closed by clicking on the FAB or by calling the JS API methods. Example usage:

Beacon('on', event, callback)
Beacon('once', event, callback)
Beacon('off', event, callback?)

Example usage

Beacon('on', 'open', () => console.log('Beacon opened'))

Beacon('on', 'close', () => console.log('Beacon closed'))

Beacon('once', 'open', () => {
  console.log('This will only get called the first time the open event is triggered')
})

// Stop listening to a specific callback
Beacon('off', 'open', callbackPreviouslyAdded)

// Stop listening to ALL callbacks for an event
Beacon('off', 'open')
